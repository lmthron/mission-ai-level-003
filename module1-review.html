Progressive Context Loading with CLAUDE.md

How It Works

Similar to Skills in Claude web, Claude Code uses a CLAUDE.md file to understand your project without loading everything at once.

  • Claude keeps lightweight metadata (name and description)
  • When your request matches, Claude loads full instructions
  • This keeps Claude Code fast while providing specialized knowledge
  • You don't need to repeat preferences in every request

How Claude Code Executes

The Execution Loop

  1. You give a natural language request - No special syntax needed
  2. Claude analyzes the request - Determines which files to read, identifies dependencies
  3. Claude creates a plan (in plan mode) - Shows exactly what it will do
  4. You approve or adjust the plan - Review for correctness, request changes if needed
  5. Claude executes step-by-step - Makes changes incrementally, asks for permissions
  6. Claude verifies results - Runs tests automatically, checks command output
  7. You review and iterate - Examine changes, test manually, request refinements

Permission System: Claude asks before writing files, running commands, committing to git

Git Integration: Works in branches, creates descriptive commits, never pushes without permission

Rewind Feature: Press Escape twice to undo changes and restore previous state

Incremental Changes: Small, reviewable changes make it easy to identify issues

Using Plan Mode Effectively

What Plan Mode Does

Press Shift+Tab to activate plan mode. Claude researches and strategizes without modifying any files.

  • Shows exactly what it will do before doing it
  • Lists files it will modify
  • Describes commands it will run
  • You can iterate on the plan before execution

‚úÖ When to Use Plan Mode

  • ‚Ä¢ Architecture decisions
  • ‚Ä¢ Complex refactoring
  • ‚Ä¢ Multi-file changes
  • ‚Ä¢ When unsure of best approach

‚ùå Not Always Necessary

  • ‚Ä¢ Simple, well-defined tasks
  • ‚Ä¢ Single file modifications
  • ‚Ä¢ Routine boilerplate generation

Why It Matters

Plan mode prevents Claude from making assumptions about approach. You verify the strategy before committing to implementation.

Best Practices from Anthropic

1. Use Research ‚Üí Plan ‚Üí Implement Workflow

Don't jump straight to implementation. Have Claude research best practices first, create a plan, then implement. This prevents assumptions and ensures you verify strategy before execution.

2. Maintain a Comprehensive CLAUDE.md

Document everything: build commands, code style, testing requirements, repository conventions, known gotchas. Use the # shortcut to add learnings as you work.

3. Always Use Version Control

Work in git branches. Claude Code encourages this and makes it easy. Safe to experiment, easy to review changes, clean history, simple rollback.

4. Leverage Plan Mode for Complex Tasks

Use Shift+Tab for architecture decisions, complex refactoring, multi-file changes, or when unsure of approach. Iterate on plans before writing code.

5. Be Specific About Verification

Instead of "Add tests," say "Add integration tests that verify actual business logic without mocking the database. Test both success and failure cases."

6. Build Project Knowledge with # Shortcut

Capture learnings as you work: # We use Zod for validation because it provides TypeScript types automatically

7. Review Changes Incrementally

Don't let Claude Code make 50 file changes before reviewing. Ask for incremental work so you can spot issues early with smaller changesets.
